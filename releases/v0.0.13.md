# v0.0.13

> Released: 2026-04-02

## Highlights

- **Level 2 context caching** — Server-side pre-compiled heartbeat context bundles eliminate redundant API calls from agents. Issues, ancestors, project info, and incremental comment deltas are assembled before adapter execution and injected into the prompt. All 6 adapters use stable prefix ordering for Ollama KV cache and Claude prompt cache hits. DB migration adds `lastSeenCommentId` for per-agent comment tracking.
- **Transient error auto-retry** — MCP timeouts, rate limits (429/503), SIGTERM (exit 143), and network errors now trigger automatic retry after 30 seconds. Agents no longer get stuck in error state from temporary failures.
- **Headless permissions overhaul** — Claude adapter defaults `dangerouslySkipPermissions` to `true`. OpenCode adapter sets all 10 permission types (bash, read, edit, glob, grep, list, lsp, skill, task, external_directory) to `allow`. Codex adapter defaults sandbox bypass to `true`. Agents can now execute `op` commands, read files, and run bash without interactive approval.

## Improvements

- **Session handoff with PARA memory** — When sessions rotate, the handoff now points agents to their persistent PARA memory files (daily notes, knowledge graph, MEMORY.md) for context recovery across session boundaries.
- **Session compaction tuning** — Balanced thresholds (50 runs, 2M tokens, 24h) prevent session overflow while maintaining sufficient history.
- **Gemini 3.x models** — Gemini adapter updated with Gemini 3.1 Pro Preview (flagship, 1M context), Gemini 3 Pro (coding/math), Gemini 3 Flash (fast, free tier). Deprecated 2.0 models removed.
- **Glob-before-read directive** — All 6 adapters inject "NEVER guess file paths, ALWAYS use glob/ls to discover" into agent instructions, preventing hallucinated path failures from smaller models.

## Fixes

- **Session handoff file safety** — Handoff no longer points to specific daily note files that may not exist; instructs agents to check with glob/ls first.
- **OpenCode cloud models** — `ollama-cloud` provider properly registered; agents can use Kimi K2.5, GLM-5, MiniMax, and other cloud models via Ollama.

## Database Migration

- **Migration 0046** — Adds `last_seen_comment_id` column to `agent_task_sessions` table for incremental comment delta tracking. Nullable text column, zero-downtime.

## Advisory

**Codex Local Adapter**: The `codex_local` adapter has significantly higher token consumption (~936K input tokens/run) compared to `claude_local` (~37/run) or `opencode_local` (~211K/run). We recommend `claude_local` or `opencode_local` with local Ollama models. See [ROADMAP.md](../ROADMAP.md).
