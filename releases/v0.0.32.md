# v0.0.32

> Released: 2026-04-17

## Fixes (all verified live on fresh AWS EC2 instances)

- **Installer crashed mid-install on fresh Linux / Mac** — `set -u` in
  `install.sh` and `install-linux.sh` aborted the install when `nvm.sh`
  sourced `PROVIDED_VERSION` (unset during `nvm use`). Confirmed live on
  Ubuntu 24.04 EC2 — installer got to `nvm install --lts`, then exited before
  `npm install -g fidelios` or `fidelios onboard`. Same pattern affected the
  Homebrew / `brew shellenv` path on macOS. Dropped `set -u`.
- **`fidelios onboard --yes` left an orphaned server bound to port 3100** —
  `--yes` alone used to imply `--run`, so the one-liner installer
  (`curl | bash` → `fidelios onboard --yes`) spun up a background server that
  blocked the user's first manual `fidelios run`. `--yes` now only auto-starts
  the server when stdin + stdout are TTYs (interactive use). Pass `--run`
  explicitly if you want the old auto-start behaviour.
- **Linux installer now installs Node.js + CLI** instead of Docker — single
  unified install story across macOS / Linux / Windows, no dependency on a
  public container registry, and users get the full `fidelios` CLI.

## Docs

- Rewrote [docs/start/install/linux.md](https://docs.fidelios.nl/start/install/linux)
  and [docs/start/install/cloud-vms.md](https://docs.fidelios.nl/start/install/cloud-vms)

## Under the hood

Verified end-to-end on Ubuntu 24.04 EC2 (`t3.small`, fresh AMI). Install took
~60 seconds on first try after these fixes; second manual `fidelios run` works
without "port already in use".
