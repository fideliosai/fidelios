# v0.0.39

> Released: 2026-04-21
> Telegram Gateway plugin: **Test Connection** and **Create Topics** buttons now activate after you save your bot token.

## What's fixed

After saving a valid bot token and chat ID in the Telegram Gateway
plugin settings, the custom settings page kept showing:

> Not configured — set botToken and chatId in plugin settings

and both **Test Connection** and **Create Topics** buttons stayed
permanently disabled — no matter how many times you re-saved the
configuration.

## Root cause

Two pieces of logic at war:

- The worker's `plugin-status` data handler deliberately strips the
  bot token before sending config to the browser. Good: don't leak
  secrets into the UI.
- But the UI then checked `config.botToken && config.chatId` to decide
  whether the plugin was configured. Since `botToken` was never in the
  payload, this check was always false — buttons disabled, badge stuck
  on "Not configured", forever.

## Fix

- Worker returns a `hasBotToken: boolean` flag in the status payload.
  The raw token never leaves the server.
- UI checks `config.hasBotToken && config.chatId` — flips to green
  **Configured** as soon as you save a real token.
- Reworded the stuck badge message: it now points you at the form
  below the custom page (which was the source of the original UX
  confusion — the old copy said "set these in plugin settings" when
  you were already in plugin settings).
- Bumped the plugin manifest version 0.1.0 → 0.1.1 so you can tell
  from the header which build you have.

## How to pick up the fix

```
npm install -g fidelios@latest
```

Then in FideliOS → Settings → Plugins → Telegram Gateway:

1. If you already had the plugin installed, uninstall and re-install
   the example to pull in v0.1.1.
2. Fill in the form (Bot Token, Chat ID, Default Topic ID) and click
   **Save Configuration**.
3. The orange badge flips to green **Configured**, and Test
   Connection + Create Topics become clickable.

## Security note for anyone testing this flow

If you pasted a bot token into the old form and shared a screenshot
while debugging, treat that token as compromised. Revoke it in
[@BotFather](https://t.me/BotFather) with `/revoke` and paste the new
one into the form.
