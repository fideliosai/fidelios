# v0.0.33

> Released: 2026-04-17
> All three installers verified end-to-end on fresh AWS EC2 (Ubuntu 24.04,
> Windows Server 2022, macOS 26.4 arm64).

## Installer fixes (all live-verified on fresh cloud instances)

- **macOS / Linux**: the installer now writes the nvm loader block into
  `~/.zshrc` + `~/.zprofile` (or `~/.bashrc` + `~/.bash_profile` for bash
  users) when Node.js is installed via nvm. Without this, a fresh macOS
  with zsh as default shell (every modern Mac) left `fidelios` unreachable
  after reopening Terminal — nvm's own installer only touches `.bashrc`
  which zsh doesn't read. Verified on a truly-clean `mac2-m2.metal` EC2.
- **Windows**: corrected nvm-windows silent install flags
  (`/VERYSILENT /SUPPRESSMSGBOXES /NORESTART /SP-`) and invoke `nvm.exe`
  via its full path so downstream `node` / `npm` calls resolve without
  needing a new PowerShell session. Confirmed on Windows Server 2022 SSM.
- **`fidelios onboard --yes` no longer auto-starts the server in non-TTY
  installs.** The one-liner `curl | bash` path used to leave an orphaned
  server bound to port 3100, blocking the user's first manual `fidelios
  run`. Non-TTY installs now only create the config; pass `--run`
  explicitly if you want the old auto-start behaviour.
- **`install-linux.sh`** is now unified with `install.sh` (Node+CLI path).
  Docker is still available for container deployments via the published
  `ghcr.io/fideliosai/fidelios` image, but is no longer the default for
  single-VM installs — you get the full `fidelios` CLI either way.
